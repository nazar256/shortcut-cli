# Security policy

## Supported versions

Security fixes are expected to land on `main` first and ship in the next or latest supported release.

Older releases may not receive fixes.

## Reporting a vulnerability

If you believe you found a security issue, please avoid opening a public issue first.

Preferred path:

1. use GitHub private vulnerability reporting / security advisories when it is enabled for this repository

Private reporting should be enabled before public launch. Until then, please do not publish sensitive details in a public issue.

For non-sensitive hardening suggestions, a normal GitHub issue is fine.
